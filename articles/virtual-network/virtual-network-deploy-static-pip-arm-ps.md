## <a name="step-1---start-your-script"></a>Stap 1: beginnen uw script

U kunt de volledige PowerShell-script gebruikt downloaden [hier](https://raw.githubusercontent.com/Azure/azure-quickstart-templates/master/IaaS-Story/03-Static-public-IP/virtual-network-deploy-static-pip-arm-ps.ps1). Volg de onderstaande stappen voor het wijzigen van het script om te werken in uw omgeving.

1. Wijzig de waarden van de variabelen op basis van de waarden die u wilt gebruiken voor uw implementatie. De waarden onder toewijzen aan het scenario gebruikt in dit document.

        # Set variables resource group
        $rgName                = "IaaSStory"
        $location              = "West US"
        
        # Set variables for VNet
        $vnetName              = "WTestVNet"
        $vnetPrefix            = "192.168.0.0/16"
        $subnetName            = "FrontEnd"
        $subnetPrefix          = "192.168.1.0/24"
        
        # Set variables for storage
        $stdStorageAccountName = "iaasstorystorage"
        
        # Set variables for VM
        $vmSize                = "Standard_A1"
        $diskSize              = 127
        $publisher             = "MicrosoftWindowsServer"
        $offer                 = "WindowsServer"
        $sku                   = "2012-R2-Datacenter"
        $version               = "latest"
        $vmName                = "WEB1"
        $osDiskName            = "osdisk"
        $nicName               = "NICWEB1"
        $privateIPAddress      = "192.168.1.101"
        $pipName               = "PIPWEB1"
        $dnsName               = "iaasstoryws1"

## <a name="step-2---create-the-necessary-resources-for-your-vm"></a>Stap 2: de benodigde resources voor uw VM maken

Voordat u een VM maakt, moet u een resourcegroep, VNet openbare IP- en NIC moet worden gebruikt door de VM.

1. Maak een nieuwe resourcegroep.

        New-AzureRmResourceGroup -Name $rgName -Location $location
        
2. De VNet en subnet maken.

        $vnet = New-AzureRmVirtualNetwork -ResourceGroupName $rgName -Name $vnetName `
            -AddressPrefix $vnetPrefix -Location $location   
        
        Add-AzureRmVirtualNetworkSubnetConfig -Name $subnetName `
            -VirtualNetwork $vnet -AddressPrefix $subnetPrefix
        
        Set-AzureRmVirtualNetwork -VirtualNetwork $vnet 

3. Maak het openbare IP-resource. 

        $pip = New-AzureRmPublicIpAddress -Name $pipName -ResourceGroupName $rgName `
            -AllocationMethod Static -DomainNameLabel $dnsName -Location $location

4. De netwerkinterface (NIC) voor de VM in de bovenstaande gemaakt met het openbare IP-subnetten maken. Zoals u ziet, de eerste cmdlet voor het ophalen van de VNet van Azure, dit is nodig aangezien een **Set-AzureRmVirtualNetwork** is uitgevoerd als u wilt wijzigen van de bestaande VNet.

        $vnet = Get-AzureRmVirtualNetwork -Name $vnetName -ResourceGroupName $rgName
        $subnet = Get-AzureRmVirtualNetworkSubnetConfig -VirtualNetwork $vnet -Name $subnetName
        $nic = New-AzureRmNetworkInterface -Name $nicName -ResourceGroupName $rgName `
            -Subnet $subnet -Location $location -PrivateIpAddress $privateIPAddress `
            -PublicIpAddress $pip

5. Maak een account opslag als host voor het station VM OS.

        $stdStorageAccount = New-AzureRmStorageAccount -Name $stdStorageAccountName `
            -ResourceGroupName $rgName -Type Standard_LRS -Location $location

## <a name="step-3---create-the-vm"></a>Stap 3: de VM maken 

Nu alle benodigde resources zijn op hun plaats staan, kunt u een nieuwe VM maken.

1. Maak het configuratieobject voor VM.

        $vmConfig = New-AzureRmVMConfig -VMName $vmName -VMSize $vmSize 

1. Referenties voor het lokale beheerdersaccount VM ophalen.

        $cred = Get-Credential -Message "Type the name and password for the local administrator account."

2. Een VM configuratie-object maken.

        $vmConfig = Set-AzureRmVMOperatingSystem -VM $vmConfig -Windows -ComputerName $vmName `
            -Credential $cred -ProvisionVMAgent -EnableAutoUpdate

3. Afbeelding van het besturingssysteem instellen voor de VM.

        $vmConfig = Set-AzureRmVMSourceImage -VM $vmConfig -PublisherName $publisher `
            -Offer $offer -Skus $sku -Version $version

4. Configureer de schijf OS.

        $osVhdUri = $stdStorageAccount.PrimaryEndpoints.Blob.ToString() + "vhds/" + $osDiskName + ".vhd"
        $vmConfig = Set-AzureRmVMOSDisk -VM $vmConfig -Name $osDiskName -VhdUri $osVhdUri -CreateOption fromImage

5. De NIC toevoegen aan de VM.

        $vmConfig = Add-AzureRmVMNetworkInterface -VM $vmConfig -Id $nic.Id -Primary

6. Maak de VM.

        New-AzureRmVM -VM $vmConfig -ResourceGroupName $rgName -Location $location

7. Sla het scriptbestand.
